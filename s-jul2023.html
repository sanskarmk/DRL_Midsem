7.5 marksQuestion 1 · Zebra-crossing traffic signal

(a) MDP formulation + design choices. [3.5]

▶Show solution
  • States: \(S=(T,P,V)\) — signal state \(T\in\{\text{Red},\text{Green}\}\), number of waiting pedestrians \(P\), waiting vehicles \(V\).
  • Actions: {switch to Red, switch to Green, maintain}.
  • Transitions: depend on action + arrival rates (pedestrians/vehicles accumulate or clear); estimated from traffic data.
  • Reward: \(R=-(\alpha P+\beta V)\) — penalise total waiting; weights \(\alpha,\beta\) trade pedestrian vs vehicle delay.
  • Discount: \(\gamma\approx0.9\) to value sustained low waiting, not just the next step.

(b) Bellman state-value equation + backup diagram. [2]

▶Show solution
\[ v_\pi(s)=\sum_a\pi(a\mid s)\sum_{s',r}p(s',r\mid s,a)\big[r+\gamma v_\pi(s')\big] \]

(c) Which solution approach is appropriate? [1]

▶Show solution

The full model is hard to specify exactly and the state space \((T,P,V)\) grows large with queue lengths, so tabular DP is impractical. A model-free / function-approximation approach (TD learning, or DQN for large state spaces) that learns from interaction is appropriate. If you discretise \(P,V\) coarsely and can estimate transitions, small-scale DP (value iteration) also works.

(d) Multiply all rewards by 10. [1]

▶Show solution
Answer Every value scales: \(v'(s)=10\,v(s)\). The optimal policy is unchanged — a positive scaling preserves the ordering of action values, and the policy depends only on which action is largest.

(c) Compare (a) and (b). [1.5]

▶Show solution

Both flag t4, t5. They diverge afterward: with sample averages estimates are smooth (equal weight), so c stays below b until late ⇒ extra exploratory picks at t9, t12. With constant α, recent rewards dominate, so b collapses quickly (to 1.625 by t7) ⇒ the exploratory pick shifts to t7. The constant-α estimates are more volatile, changing which picks look non-greedy.

(d) Initialise all action values to 5. [1.5]

▶Show solution

Optimistic initialisation. Every genre now starts above any possible reward (max 3), so each first looks great and the agent is driven to try them all early — extra exploration up front even with the same ε. As samples arrive the inflated values decay toward the true means; afterward behaviour is the same. So early exploratory moves increase; the effect is temporary.

(e) Extend to n individuals with different tastes. [1.5]

▶Show solution

Keep a separate bandit per user, or use a contextual bandit where the user’s identity/features are the context and \(Q(\text{context},a)\) is learned. Share learning across similar users and use optimistic initial values to handle cold-start.

7.5 marksQuestion 3 · Robot line-world (positions 0–5)

States 0–5; 0 and 5 terminal. Left ⇦ → s−1, Right ⇒ → s+1. Entering 5 gives +x, entering 0 gives −x, else 0. \(x=\text{(last ID digit)}+5\). γ = 0.5, in-place DP, sweep order 1→4.

(a) Evaluate policy [⇦ ⇦ ⇦ ⇒], then improve (one iteration). [3]

▶Show solution

Evaluation (V init 0; in-place uses freshly-updated neighbours):

  1. \(V(1)\): π=⇦ → state 0, r=−x. \(V(1)=-x+0.5\,V(0)=-x\).
  2. \(V(2)\): π=⇦ → state 1, r=0. \(V(2)=0+0.5\,V(1)=0.5(-x)=-x/2\).
  3. \(V(3)\): π=⇦ → state 2, r=0. \(V(3)=0+0.5\,V(2)=-x/4\).
  4. \(V(4)\): π=⇒ → state 5, r=+x. \(V(4)=x+0.5\,V(5)=x\).

So \(V=(V_1,V_2,V_3,V_4)=(-x,\,-x/2,\,-x/4,\,x)\).

Improvement (greedy w.r.t. these V, for \(x>0\)):

state⇦ value⇒ valuegreedy
1−x0.5·V(2) = −x/4⇒
20.5·V(1) = −x/20.5·V(3) = −x/8⇒
30.5·V(2) = −x/40.5·V(4) = x/2⇒
40.5·V(3) = −x/8x⇒
Improved policy [⇒ ⇒ ⇒ ⇒] — head right toward the +x terminal from every state. (Intuition: the +x goal is at the right; left only risks the −x pit.)

(b) Two iterations of value iteration. [3]

▶Show solution

\(V(s)\leftarrow\max(\,⇦:\,r+0.5V(s-1),\;\;⇒:\,r+0.5V(s+1)\,)\), in-place order 1→4, V init 0.

Iteration 1:

  1. \(V(1)=\max(-x+0,\;0+0.5\cdot0)=\max(-x,0)=0\)
  2. \(V(2)=\max(0.5\cdot0,\;0.5\cdot0)=0\)
  3. \(V(3)=\max(0.5\cdot0,\;0.5\cdot0)=0\)
  4. \(V(4)=\max(0.5\cdot0,\;x+0)=x\)

After iter 1: \(V=(0,0,0,x)\).

Iteration 2:

  1. \(V(1)=\max(-x,\;0.5\,V(2)=0)=0\)
  2. \(V(2)=\max(0.5\,V(1)=0,\;0.5\,V(3)=0)=0\)
  3. \(V(3)=\max(0.5\,V(2)=0,\;0.5\,V(4)=x/2)=x/2\)
  4. \(V(4)=\max(0.5\,V(3)=x/4,\;x)=x\)
After 2 iterations \(V=(0,\,0,\,x/2,\,x)\). The +x signal propagates one state left per sweep — value iteration needs several sweeps for it to reach state 1.

(c) Applicability to large state/action spaces. [1.5]

▶Show solution

Both algorithms sweep every state each iteration and need the full model, so cost scales with \(|\mathcal S|\times|\mathcal A|\) and they become intractable for large/continuous spaces (curse of dimensionality). Policy iteration may need fewer iterations but each evaluation is itself expensive; value iteration is cheaper per sweep but may need many sweeps. For large problems, switch to sampling + function approximation (TD, DQN).

7.5 marksQuestion 4 · Multi-user Monte Carlo
⛔
Partly out of scope for your mid-sem
Part (a) (first/every-visit MC) is in scope. Parts (b) (importance-sampling ratio) and (d) (off-policy coverage) cover off-policy MC / importance sampling, which your professor said is not in the mid-sem — kept here for completeness only.

Individuals {p,q,r}, genres {a,b,c}. Episodes (individual, genre, reward), target policy π(p)=a, π(q)=a, π(r)=c:

E1: p,a,0  q,c,2  p,b,0  q,c,3  r,c,2  r,b,2  q,c,1
E2: q,b,3  r,a,2  q,c,1  p,a,0  q,a,2  p,c,2  r,b,1
E3: r,a,2  p,b,1  p,a,2  q,c,3  q,b,3  r,a,3  q,c,0

(a) First-visit & every-visit prediction; which converges faster? [3]

▶Show solution
📝
Interpretation (the question is loosely worded): treat each individual as a state, γ = 1, and the return at a visit = sum of that episode’s rewards from that point onward.

First-visit (first time the individual appears in each episode):

  • p: E1 = 0+2+0+3+2+2+1 = 9; E2 (first p at pos4) = 0+2+2+1 = 5; E3 (first p at pos2) = 1+2+3+3+3+0 = 12 → mean = (9+5+12)/3 = 9.0
  • q: E1 (pos2) = 2+0+3+2+2+1 = 10; E2 (pos1) = 3+2+1+0+2+2+1 = 11; E3 (pos4) = 3+3+3+0 = 9 → mean = (10+11+9)/3 = 10.0
  • r: E1 (pos5) = 2+2+1 = 5; E2 (pos2) = 2+1+0+2+2+1 = 8; E3 (pos1) = 2+1+2+3+3+3+0 = 14 → mean = (5+8+14)/3 = 9.0

Every-visit (all appearances) gives V(p) ≈ 8.17, V(q) ≈ 6.22, V(r) ≈ 5.67.

Which converges faster? Every-visit extracts more returns per episode, so with limited data it produces estimates sooner; first-visit returns are independent ⇒ unbiased and cleaner. Both converge to \(v_\pi\) as episodes → ∞.

(b) Importance-sampling ratio for E2; relation to model dynamics. [2]

▶Show solution

With a uniform behaviour policy \(b(\cdot\mid s)=1/3\) and the deterministic target π, the ratio is

\[ \rho_{E2}=\prod_k\frac{\pi(A_k\mid S_k)}{b(A_k\mid S_k)} \]

E2 begins with (q, b), but \(\pi(b\mid q)=0\) (target wants a). One zero factor ⇒

ρ(E2) = 0 E2 contains actions the target policy would never take, so under ordinary importance sampling it contributes nothing to the estimate of \(v_\pi\).

Relation to model dynamics: the trajectory probability under each policy contains the same environment factors \(p(S_{k+1}\mid S_k,A_k)\); they appear in both numerator and denominator and cancel. Hence \(\rho\) depends only on the policies, not the (possibly unknown) model — which is what makes off-policy MC model-free.

(c) Two approaches that keep MC exploring. [1.5]

▶Show solution

Exploring starts (random initial (s,a)) and on-policy ε-soft policies (every action keeps probability ≥ ε/|𝒜|). Off-policy learning via a soft behaviour policy is a third.

(d) Must off-policy methods have non-zero probabilities for all actions? [1]

▶Show solution
Coverage assumption — yes The behaviour policy must satisfy \(b(a\mid s)>0\) wherever \(\pi(a\mid s)>0\) (else the IS ratio divides by zero and those actions are never sampled). The target policy itself may be deterministic; \(b\) just has to cover everything π might do.
