5 marksQuestion 1 · Rewards, TD & algorithm types

(a) How are rewards and returns connected? [1]

▶Show solution

The return is the (discounted) sum of future rewards: \(G_t=R_{t+1}+\gamma R_{t+2}+\gamma^2R_{t+3}+\cdots=\sum_{k\ge0}\gamma^k R_{t+k+1}\), with the recursion \(G_t=R_{t+1}+\gamma G_{t+1}\). A reward is the one-step signal; the return aggregates rewards over time — and it is the return (via value) that the agent maximises.

(b) TD for tic-tac-toe: can α encourage exploration? [3]

▶Show solution
No α is the step size in \(V(S_t)\leftarrow V(S_t)+\alpha[R_{t+1}+\gamma V(S_{t+1})-V(S_t)]\).

Why not: α only controls how fast value estimates move toward the TD target — it changes learning speed and stability, not which actions are tried. Exploration is a property of the action-selection rule (e.g. ε-greedy / softmax), which decides whether to take a non-greedy move. Turning α up or down never makes the agent visit new states; it just re-weights updates for the states it already chooses to visit.

(c) One model-based and one model-free algorithm. [1]

▶Show solution

Model-based: Value Iteration (or Policy Iteration). Model-free: Monte-Carlo control (or TD / Q-learning).

5 marksQuestion 2 · Exploration & bandit framing

(a) No explicit exploration ⇒ is RL the same as supervised learning? [2.5]

▶Show solution
No

(1) Even without explicit exploratory moves, the feedback is still evaluative (a reward saying how good the move was), not a supervised label giving the correct move — so the learning signal differs fundamentally. (2) The agent’s own greedy actions still determine the data distribution it sees (its moves change future states), whereas supervised learning is handed a fixed, independent labelled dataset. So it remains RL, just with poor exploration.

(b) A situation where greedy beats ε-greedy. [1]

▶Show solution

When rewards are deterministic and stationary: one trial reveals each arm’s exact value, so any further exploration is wasted reward — pure greedy is optimal. (Also when there are very few steps left to benefit from new knowledge.)

(c) A workplace MAB use-case. [1.5]

▶Show solution

Email subject-line / CTA testing. Arms = candidate subject lines; action = which line to send to the next recipient; reward = open/click (1/0); the decision recurs per recipient and doesn’t change future recipients’ state. Goal = maximise total opens. (Any “pick-one-of-k-options, observe reward, repeat, no state change” task qualifies — ad placement, model/endpoint selection, pricing tiers.)

5 marksQuestion 3 · Non-stationary MAB, exploration tricks

(a) Using the learning rule for a non-stationary MAB. [2]

▶Show solution

Use a constant step size: \(Q_{n+1}=Q_n+\alpha[R_n-Q_n]\), \(\alpha\in(0,1]\). (1) This is an exponential recency-weighted average — recent rewards get weight \(\alpha\), older ones decay by \((1-\alpha)\) per step, so the estimate tracks a value that drifts over time. (2) Unlike the \(1/n\) sample average (which converges and then stops adapting), constant α never stops adapting, which is exactly what a non-stationary problem needs.

(b) Optimistic Initial Values & UCB — when (not) to use. [2]

(c) What info do you need to find the optimal value function? [1]

▶Show solution

The complete MDP: states, actions, the dynamics \(p(s',r\mid s,a)\) (transition probabilities and rewards), and the discount \(\gamma\). With those, value/policy iteration computes \(v_*\).

5 marksQuestion 4 · Recycling robot & maze rewards

(a) Write qπ(high, search) and q*(high, search). [2]

▶Show solution

From high, search stays high w.p. α (reward \(r_\text{search}\)) or drops to low w.p. 1−α (reward \(r_\text{search}\)):

\[ q_\pi(\text{high},\text{search})=r_\text{search}+\gamma\big[\alpha\,v_\pi(\text{high})+(1-\alpha)\,v_\pi(\text{low})\big] \]
\[ q_*(\text{high},\text{search})=r_\text{search}+\gamma\big[\alpha\,v_*(\text{high})+(1-\alpha)\,v_*(\text{low})\big] \]

with \(v_*(s)=\max_{a}q_*(s,a)\). Equivalently \(q_*(\text{high},\text{search})=r_\text{search}+\gamma[\alpha\max_{a'}q_*(\text{high},a')+(1-\alpha)\max_{a'}q_*(\text{low},a')]\).

(b) Maze: +1 for escape, 0 otherwise — no improvement. What’s wrong? Fix? [3]

▶Show solution

What’s wrong. With reward +1 only at escape and no discounting (or γ=1, undiscounted episodic), every policy that eventually escapes earns the same total return of +1 — whether it takes 10 steps or 10,000. So there is no signal that distinguishes a fast escape from a slow one; the agent has nothing to improve and shows no progress.

How to fix (either works):

  • Discount the +1: use γ < 1. Then escaping in fewer steps yields a larger return \(\gamma^{\,T}\), giving the agent a reason to escape sooner.
  • Reshape rewards: give −1 per time step (and 0 at escape), so the agent maximises return by minimising steps to the exit.
