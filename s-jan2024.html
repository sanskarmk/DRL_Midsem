Five questions spanning the elements of RL & the TD rule, bandit ε-greedy, MDP returns, policy iteration, and Monte-Carlo value estimation.

6 marksQuestion 1 · Tic-tac-toe, elements & TD

(a) Four elements of RL + the role of each. [2]

▶Show solution
  1. Policy — a mapping \(\pi(a\mid s)\) from states to actions; it defines the agent’s behaviour (which move to make in a board position).
  2. Reward signal — a scalar received each step defining the goal (e.g. +1 win, −1 loss, 0 otherwise); the agent cannot change it, only act to maximise it.
  3. Value function — \(v_\pi(s)\)/\(q_\pi(s,a)\): the expected long-run return from a state (or state-action); it predicts total future reward, guiding move selection beyond immediate reward.
  4. Model (optional) — \(p(s',r\mid s,a)\): the agent’s representation of the environment used to plan ahead; methods without it are model-free.

See Topic ① · elements.

(b) Temporal-difference rule for a state value + explain. [2]

▶Show solution
\[ V(S_t)\leftarrow V(S_t)+\alpha\big[\,R_{t+1}+\gamma V(S_{t+1})-V(S_t)\,\big] \]
  • \(\alpha\) — step size (learning rate): fraction of the error we move.
  • \(R_{t+1}+\gamma V(S_{t+1})\) — the TD target: a bootstrapped estimate of the return using the observed reward and the current estimate of the next state.
  • \(\delta_t=R_{t+1}+\gamma V(S_{t+1})-V(S_t)\) — the TD error (“surprise”).

Working: after each move we nudge the current state’s value toward the reward-plus-discounted-next-value; positive error raises it, negative lowers it. Updates happen online, every step, without waiting for the game to end.

(c) Revise V using (b): V=4.5, successors=2.7, parameter 0.9. [2]

▶Show solution

Take \(\alpha=\gamma=0.9\) (“0.9 for any parameter”) and \(R_{t+1}=0\) (no reward stated), \(V(S_{t+1})=2.7\):

\[ V\leftarrow 4.5+0.9\big[0+0.9(2.7)-4.5\big]=4.5+0.9(2.43-4.5)=4.5+0.9(-2.07) \]
Answer \(V = 4.5-1.863 = \mathbf{2.637}\). The estimate drops because the successors (2.7) are worth far less than the optimistic current value 4.5, giving a negative TD error.
6 marksQuestion 2 · Multi-armed bandit

(a) A use-case modellable as a MAB. [2]

▶Show solution

Online ad selection. Arms = the \(k\) candidate ads; action = which ad to display to an incoming user; reward = 1 if clicked, else 0; the same decision recurs every impression and the choice does not change the next user’s situation (single state). Goal: maximise total clicks. All MAB elements present: fixed action set, stochastic reward per arm, no state transition.

(b) ε-greedy bandit pseudocode + efficiency. [2.5]

▶Show solution
Initialise, for a = 1..k:
    Q(a) ← 0 ;  N(a) ← 0
loop forever:
    A ← argmax_a Q(a)        with probability 1−ε   (exploit)
        a random action      with probability ε     (explore)
    R ← bandit(A)
    N(A) ← N(A) + 1
    Q(A) ← Q(A) + (1/N(A)) · [R − Q(A)]

Efficiency: \(O(1)\) time and memory per step (incremental update). It always eventually finds the best arm, but it is wasteful — it keeps exploring even arms already known to be poor, and never stops exploring while ε > 0.

(c) How does it balance exploration vs exploitation? [1.5]

▶Show solution

With probability \(1-\varepsilon\) it exploits (takes the current best arm, banking known reward); with probability \(\varepsilon\) it explores (samples a random arm to improve estimates). The fixed ε guarantees every arm keeps being sampled, so a genuinely-better arm is eventually discovered, while most pulls still earn from the best-known arm. Larger ε ⇒ more exploration; smaller ε ⇒ more exploitation.

6 marksQuestion 4 · Policy iteration & DP suitability

(a) What is Policy Evaluation vs Policy Improvement; use in finding π*. [2]

▶Show solution

Policy Evaluation: given a fixed policy \(\pi\), iteratively solve the Bellman expectation equation to obtain its value function \(v_\pi\) (predict how good \(\pi\) is). Policy Improvement: given \(v_\pi\), construct \(\pi'(s)=\arg\max_a\sum p(s',r\mid s,a)[r+\gamma v_\pi(s')]\), a policy that is provably no worse (policy-improvement theorem). Alternating the two — evaluation makes value consistent with policy, improvement makes policy greedy w.r.t. value — converges to the optimal pair \((v_*,\pi_*)\).

(b) High-level policy-iteration algorithm. [2]

▶Show solution
1. Initialise π arbitrarily, V(s)=0 ∀s
2. POLICY EVALUATION:
     repeat: for each s:  V(s) ← Σ p(s',r | s,π(s)) [r + γV(s')]
     until max change < θ
3. POLICY IMPROVEMENT:
     stable ← true
     for each s:
        a* ← argmax_a Σ p(s',r | s,a)[r + γV(s')]
        if a* ≠ π(s): π(s) ← a* ; stable ← false
4. if stable: return (V≈v*, π≈π*)  else go to 2

(Assumes a known model and a finite state/action space.)

(c) When is DP appropriate? Two examples. [2]

▶Show solution

DP suits problems that are Markov, have a fully known model \(p(s',r\mid s,a)\), and a small/finite state-action space that can be swept exhaustively. Examples: gambler’s problem (coin-flip dynamics known); small gridworld navigation; inventory control with a known demand distribution; the recycling robot.

(a) Estimate V(A) and V(B) from the episodes. [2]

A 2 A 6 B 1 B 0 T
A 3 B 2 A 4 B 2 B 0 T
A 0 B 2 A 4 B 4 B 2 B 0 T
B 8 B 0 T
▶Show solution
📝
The paper says “5 episodes” but lists 4 — treated as a typo; we use the 4 given. No γ is stated, so take the standard \(\gamma=1\) (undiscounted): each return = sum of rewards from that visit to T.

First-visit (first occurrence of the state in each episode):

  • A: episode-1 first-A = 2+6+1+0 = 9; ep-2 = 3+2+4+2+0 = 11; ep-3 = 0+2+4+4+2+0 = 12 → mean = (9+11+12)/3 = 32/3 ≈ 10.67
  • B: ep-1 first-B = 1+0 = 1; ep-2 = 2+4+2+0 = 8; ep-3 = 2+4+4+2+0 = 12; ep-4 = 8+0 = 8 → mean = (1+8+12+8)/4 = 29/4 = 7.25

Every-visit (all occurrences):

  • A returns: 9,7,11,6,12,10 → mean = 55/6 ≈ 9.17
  • B returns: 1,0,8,2,0,12,6,2,0,8,0 → mean = 39/11 ≈ 3.55
Answer First-visit: V(A) ≈ 10.67, V(B) = 7.25.   Every-visit: V(A) ≈ 9.17, V(B) ≈ 3.55. (If your section uses first-visit, quote the first pair; show the method either way.)

(b) MC techniques for the exploration/exploitation balance. [2]

▶Show solution
  • Exploring starts — start episodes from random (s,a) so every action gets sampled.
  • On-policy ε-soft policies — behave with a policy that gives every action probability ≥ ε/|𝒜|, guaranteeing continual exploration.
  • Off-policy learning — explore with a soft behaviour policy \(b\) while learning the greedy target \(\pi\) (via importance sampling).

(c) How are ε-soft & ε-greedy related? 4-action example. [2]

▶Show solution

ε-greedy is a special case of ε-soft. An ε-soft policy is any with \(\pi(a\mid s)\ge\varepsilon/|\mathcal A|\) for all \(a\); ε-greedy puts the leftover mass on the greedy action.

Example — state with 4 actions, greedy = a₃, ε = 0.4 ⇒ ε/|𝒜| = 0.1:

actiona₁a₂a₃ (greedy)a₄
π0.10.11−ε+ε/4 = 0.70.1

Greedy action: \(1-0.4+0.1=0.7\); each other: \(0.4/4=0.1\); total = 1. Every action keeps non-zero probability (ε-soft), with the greedy one dominant (ε-greedy).
