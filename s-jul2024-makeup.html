6 marksQuestion 1 · Concepts

(a) How do unsupervised learning and RL differ? [2]

▶Show solution

Goal & signal. Unsupervised learning finds hidden structure in unlabelled data (clusters, density, low-dim representations) with no notion of reward or goal. RL maximises a reward signal through action — it has an evaluative feedback signal and an objective. Data. Unsupervised data is a fixed set of inputs; RL data is generated by the agent’s own actions and is sequential/non-i.i.d. RL is not “unsupervised” just because it lacks labels — the reward is its supervision.

(b) Model-based vs model-free + when each is appropriate. [2]

▶Show solution

Model-based uses a known/learned model \(p(s',r\mid s,a)\) to plan (DP). Model-free learns values directly from sampled experience (MC, TD). Scenarios: (1) model known & state space small ⇒ model-based (value/policy iteration). (2) dynamics unknown or hard to model ⇒ model-free (learn by interacting). (3) model expensive to query but data cheap ⇒ model-free; data scarce but model available ⇒ model-based.

(c) A workplace use-case suited to a DP solution. [2]

▶Show solution

Warehouse inventory reordering. States = discrete stock levels; actions = reorder quantity; transitions = known demand distribution (so \(p\) is available); rewards = sales margin − holding/stockout cost; γ for future periods. Markov, fully modelled, modest state space ⇒ value/policy iteration computes the optimal reorder policy exactly. (Any small, fully-modelled Markov decision problem qualifies.)

6 marksQuestion 2 · Four-box game · in-place value iteration

States A, B, C, D and Goal (value 100). Edges show the immediate reward; outgoing actions are equiprobable but we apply value iteration (max over the outgoing actions). γ = 0.9, values init 0 (Goal fixed at 100), sweep order D, A, C, B, one iteration.

▶Show solution

Update \(V(s)=\max_{\text{edges}}\big[r+0.9\,V(\text{next})\big]\), reading the latest values (in-place).

  1. D (edges → A:22, C:16, Goal:40): \(\max(22{+}0.9(0),\,16{+}0.9(0),\,40{+}0.9(100))=\max(22,16,130)=\mathbf{130}\) → Goal.
  2. A (→ D:13, B:−10), using V(D)=130: \(\max(13{+}0.9(130),\,-10{+}0.9(0))=\max(130,-10)=\mathbf{130}\) → D.
  3. C (→ D:43, B:−15), using V(D)=130: \(\max(43{+}0.9(130),\,-15{+}0.9(0))=\max(160,-15)=\mathbf{160}\) → D.
  4. B (→ A:50, C:30), using V(A)=130, V(C)=160: \(\max(50{+}0.9(130),\,30{+}0.9(160))=\max(167,174)=\mathbf{174}\) → C.
Values after one in-place sweep V(D)=130, V(A)=130, V(C)=160, V(B)=174.   Policy at B B → C (since 174 > 167). In-place ordering matters: B benefits from the already-updated V(A) and V(C) in the same sweep.
6 marksQuestion 3 · Reward signs & 5-armed bandit

(a) Gridworld: do reward signs matter, or only the intervals? [2]

▶Show solution

For the optimal policy, only the relative differences (intervals) matter, not the absolute signs — provided you add the same constant to every reward in an episodic task: shifting all rewards by a constant shifts every policy’s return equally, so the arg-max policy is unchanged. Caveat: in continuing tasks (or when only some rewards are shifted) a constant offset changes the infinite discounted sum unevenly and can change behaviour. So: intervals govern the policy; signs alone don’t — but be careful with continuing tasks.

6 marksQuestion 4 · MDP example, recursive q, discounting

(a) One MDP example task. [1]

▶Show solution

Elevator dispatch: states = floor + pending call buttons; actions = move up / down / open doors; reward = −(total passenger wait time). Markov, sequential, clearly state-dependent future.

(b) Recursive expression for qπ(s,a) + significance. [2]

▶Show solution
\[ q_\pi(s,a)=\sum_{s',r}p(s',r\mid s,a)\Big[r+\gamma\sum_{a'}\pi(a'\mid s')\,q_\pi(s',a')\Big] \]

Significance: \(q_\pi\) is defined in terms of itself at successor states — this recursion (the Bellman equation) means we needn’t sum infinite future rewards directly; we can solve a set of consistent linear equations or iterate them (DP/TD). The self-referential structure is exactly what makes bootstrapping and dynamic programming possible.

(c) Continuing MDP: optimal policy for γ = 0.7 and γ = 0.5. [3]

Top state, two actions. left: reward +1, then cycles back (0) ⇒ reward stream 1,0,1,0,…   right: reward 0, then +2 on return ⇒ stream 0,2,0,2,…

▶Show solution

Value of the top state under each (period-2) policy:

\[ v_{\text{left}}=1+\gamma^2+\gamma^4+\cdots=\frac{1}{1-\gamma^2},\qquad v_{\text{right}}=2\gamma+2\gamma^3+\cdots=\frac{2\gamma}{1-\gamma^2} \]

Compare: right beats left ⟺ \(2\gamma>1\) ⟺ \(\gamma>0.5\).

  1. γ = 0.7: \(v_{\text{left}}=\dfrac{1}{1-0.49}=1.96\), \(v_{\text{right}}=\dfrac{1.4}{0.51}=2.75\). Since \(2(0.7)=1.4>1\), πright is optimal.
  2. γ = 0.5: \(v_{\text{left}}=\dfrac{1}{0.75}=1.33\), \(v_{\text{right}}=\dfrac{1}{0.75}=1.33\). Here \(2(0.5)=1\), so the two are equal — both policies are optimal (the agent is indifferent; γ = 0.5 is the break-even discount).
Answer γ=0.7 ⇒ go right (patience pays: the larger but later +2 dominates). γ=0.5 ⇒ tie, both left and right are optimal.
6 marksQuestion 5 · On-policy first-visit MC control (ε-soft)

States {A,B,C,T}, actions {x,y,z}, γ = 0.9, init Q = 0. Episodes:

E1: C x(0) A x(+10) C x(+2) B y(+12) C x(+20) T
E2: A x(+10) C x(+2) B y(+12) B y(+12) C x(+20) T
E3: C x(+2) B y(+12) C x(+20) T
▶Show solution

Returns (γ=0.9, backward); first visit marked ★:

  • E1: (C,x)=20 ; (B,y)=12+0.9(20)=30 ; (C,x)=2+0.9(30)=29 ; (A,x)★=10+0.9(29)=36.1 ; (C,x)★=0+0.9(36.1)=32.49. First-visit (C,x) is the first step ⇒ 32.49; (A,x)★=36.1.
  • E2: (C,x)=20 ; (B,y)=12+0.9(20)=30 ; (B,y)=12+0.9(30)=39 ; (C,x)=2+0.9(39)=37.1 ; (A,x)★=10+0.9(37.1)=43.39. First-visit (C,x)★=37.1, (B,y)★=39, (A,x)★=43.39.
  • E3: (C,x)=20 ; (B,y)★=12+0.9(20)=30 ; (C,x)★=2+0.9(30)=29.

Average the first-visit returns:

(s,a)first-visit returnsQ = mean
