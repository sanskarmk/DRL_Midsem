1. The agent–environment loop

RL is learning what to do — how to map situations to actions — so as to maximise a numerical reward over time. Nobody tells the agent which action is correct; it must discover which actions pay off by trying them.

At each time step \(t\): the agent sees a state \(S_t\), picks an action \(A_t\); the environment responds with a reward \(R_{t+1}\) and a next state \(S_{t+1}\). This produces the trajectory

\[ S_0,\,A_0,\,R_1,\,S_1,\,A_1,\,R_2,\,S_2,\,A_2,\,R_3,\dots \]
💡
The one-sentence intuition
RL is trial-and-error learning with delayed, evaluative feedback. “Evaluative” = the reward tells you how good the action was, not what the right action was. “Delayed” = the payoff of an action may only show up many steps later.

2. RL vs. supervised vs. unsupervised learning

This comparison appears on almost every paper (Dec 2025 Q1a, Jul 2024 Q1/Q2, Jul 2024-makeup Q1a). Memorise the table.

AspectSupervisedUnsupervisedReinforcement
Feedback signalCorrect label for every input (instructive)None — no targets at allScalar reward (evaluative, possibly delayed)
DataFixed i.i.d. (input, label) pairsInputs onlyGenerated by the agent’s own actions; not i.i.d.
GoalGeneralise the input→label mapFind hidden structure / clustersMaximise long-run cumulative reward
Key challengeOverfittingNo ground truth to scoreExploration vs. exploitation; credit assignment
🔑
Why the wristband / tic-tac-toe is RL, not supervised
There is no labelled “correct mode/move” for each situation — only a reward that arrives after acting, and an action today can change which situations you face tomorrow. Supervised learning needs the correct answer for each input up front; here we never get it. (This is exactly the answer to Dec 2025 Q1a and Jul 2024-makeup Q1a.)

vs. Unsupervised: a common trap. RL is not unsupervised just because it has no labels — it does have a feedback signal (the reward). Unsupervised learning is about finding structure in data; RL is about maximising a reward signal.

3. The four elements of RL

Sutton & Barto name four ingredients beyond the agent and environment. This is the literal answer to Jan 2024 Q1a.

Element 1

Policy  \(\pi\)

The agent’s behaviour — a mapping from states to (probabilities of) actions, \(\pi(a\mid s)\). It defines what to do. This is the thing we ultimately want to optimise.

Element 2

Reward signal  \(R\)

A single number per step defining the goal. It signals what is good immediately. The agent cannot alter it; it can only act to obtain more of it.

Element 3

Value function  \(v_\pi, q_\pi\)

The long-run desirability of a state (or state–action): expected cumulative future reward. Rewards are immediate; values are predictions of total reward to come.

Element 4

Model  \(p(s',r\mid s,a)\)

(Optional.) The agent’s representation of how the environment behaves — transition and reward dynamics. Used for planning. Methods without one are model-free.

⚖️
Reward vs. value — the distinction examiners love
Reward = immediate, short-term signal received on one transition. Value = total reward expected from now into the future. A move with low immediate reward can have high value (it sets up a win); a tempting high-reward move can have low value (it leads to a dead end). We choose actions on value, not on reward.

4. Reward, return & discounting

“Maximise reward” really means maximise the return \(G_t\) — the (discounted) sum of all future rewards:

\[ G_t \;=\; R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots \;=\; \sum_{k=0}^{\infty}\gamma^{k}R_{t+k+1} \]

The recursive form is the workhorse behind every Bellman equation:

\[ G_t = R_{t+1} + \gamma\,G_{t+1} \]

The discount factor \(\gamma\in[0,1]\) weights the future. \(\gamma\to 0\): myopic — only the next reward matters. \(\gamma\to 1\): far-sighted — distant rewards count almost as much as immediate ones. It also keeps the sum finite for continuing tasks.

💡
Why discount? Two intuitions
(1) Uncertainty — a reward you might get 50 steps from now is less certain, so weight it less. (2) Math — for a never-ending task, an undiscounted sum of rewards would be infinite and impossible to compare; \(\gamma<1\) makes it converge.

Worked numerical — computing a return

🧮
Example (this is Jan 2024 Q3b)
Episodic task, \(T=5\), rewards \(R_1{=}1, R_2{=}2, R_3{=}6, R_4{=}3, R_5{=}2\), \(\gamma=0.8\). Find \(G_0\) and \(G_1\).
\[ G_0 = 1 + 0.8(2) + 0.8^2(6) + 0.8^3(3) + 0.8^4(2) \]
\(= 1 + 1.6 + 3.84 + 1.536 + 0.8192 = \mathbf{8.7952}\)
\[ G_1 = 2 + 0.8(6) + 0.8^2(3) + 0.8^3(2) = 2+4.8+1.92+1.024 = \mathbf{9.744} \]
Sanity check with the recursion: \(G_0 = R_1 + \gamma G_1 = 1 + 0.8(9.744) = 8.7952\) ✓
Continuing case (ii): \(R_1=2\) then all \(0\)s ⟹ \(G_0 = 2,\; G_1 = 0\).

7. The Temporal-Difference (TD) update rule

TD learning is model-free: it updates a state’s value toward a bootstrapped estimate using the very next reward and the next state’s current value.

\[ V(S_t) \leftarrow V(S_t) + \alpha\big[\underbrace{R_{t+1} + \gamma V(S_{t+1})}_{\text{TD target}} - V(S_t)\big] \]

The bracket \(\delta_t = R_{t+1}+\gamma V(S_{t+1}) - V(S_t)\) is the TD error: “how surprised was I?”. \(\alpha\) is the step size (learning rate).

💡
Reading the rule
New estimate = old estimate + a fraction \(\alpha\) of the surprise. If reality (target) exceeded my prediction, nudge the value up; if it fell short, nudge it down. TD learns online, every step, without waiting for the episode to end (unlike Monte Carlo).
🧮
Worked numerical (Jan 2024 Q1c)
\(V(S_t)=4.5\), all successors \(=2.7\), all parameters \(=0.9\) (so \(\alpha=\gamma=0.9\)), reward \(R_{t+1}=0\) (none given):
\[ V \leftarrow 4.5 + 0.9\big[0 + 0.9(2.7) - 4.5\big] = 4.5 + 0.9(2.43-4.5)=4.5+0.9(-2.07)=\mathbf{2.637} \]
The estimate falls because the successors (2.7) are worth far less than the current optimistic 4.5 — the TD error is negative.
🚫
Exam trap: can \(\alpha\) drive exploration? (Jul 2024-reg Q1b)
No. \(\alpha\) is the learning rate — it controls how fast value estimates move, not which actions are tried. Exploration is governed by the action-selection rule (e.g. ε-greedy), not by \(\alpha\). Changing \(\alpha\) changes learning speed/stability, not the breadth of behaviour.

8. Check yourself

Q1. The defining feature of the reward signal in RL (vs. a supervised label) is that it is…
the correct action for the state
evaluative and possibly delayed
always available before acting
a cluster assignment
A reward tells you how good an action was (evaluative), not which action was correct (instructive). It can also arrive many steps after the action that caused it.
Q2. With \(\gamma=0.1\), the agent is…
very myopic — almost only the next reward matters
far-sighted — distant rewards dominate
unable to learn at all
guaranteed to explore more
A small \(\gamma\) crushes future terms: \(\gamma^2=0.01,\gamma^3=0.001\)… so only the immediate reward really counts. (This is the seed of the Dec 2025 Q2d answer about a near-zero \(\gamma\) destabilising long-horizon care.)
Q3. Which pair is model-free?
Policy Iteration & Value Iteration
Monte Carlo & TD learning
Value Iteration & Q-planning
Dynamic Programming & Bellman backup
MC and TD learn from sampled experience without needing \(p(s',r\mid s,a)\). DP methods (policy/value iteration) require the model.
