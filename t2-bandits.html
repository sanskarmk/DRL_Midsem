1. The k-armed bandit problem

You face \(k\) “arms” (actions). Each pull of arm \(a\) returns a reward drawn from an unknown distribution with mean \(q_*(a)=\mathbb E[R_t\mid A_t=a]\). There is no state transition — you return to the same situation each time. Goal: maximise total reward over many pulls, which means learning the \(q_*(a)\) while still earning.

🎰
A clean use-case (Jan 2024 Q2a / Jul 2024-reg Q2c)
Ad / content selection: arms = which of \(k\) ads (or Netflix genres) to show; reward = click / rating; the same decision recurs for each impression; the goal is to maximise cumulative clicks. No action changes the next “state”, so it is a bandit, not a full MDP.

2. Bandit vs. finite MDP — the key distinction

This is exactly Dec 2025 Q1b.

Multi-Armed BanditFinite MDP
One state (or independent contexts). Your action does not change future situations or future reward distributions.Multiple states; the action influences the next state, so today’s choice changes tomorrow’s opportunities.
Only immediate reward to optimise.Must optimise the long-run return (value), via Bellman equations.
💡
Decision rule for classifying
Ask: “Does choosing this action change what happens / what I can earn next time?” No → bandit. Yes → MDP. In Dec 2025 Q1, Case 1 (mode choice doesn’t affect the future) is a bandit; Case 2 (Mode C raises battery temperature, lowering future rewards for power-hungry modes) introduces state-dependence, so it becomes an MDP.

3. Estimating action values \(Q(a)\)

The natural estimate of \(q_*(a)\) is the sample average of rewards seen so far from arm \(a\):

\[ Q_n(a)=\frac{R_1+R_2+\cdots+R_{n-1}}{n-1} \]

Computed incrementally (no need to store all rewards):

\[ Q_{n+1}=Q_n+\frac{1}{n}\big[R_n-Q_n\big] \]

This has the universal RL shape NewEstimate ← OldEstimate + StepSize·(Target − OldEstimate).

4. Sample-average vs. constant step-size (the α question)

Replace the shrinking \(1/n\) with a fixed \(\alpha\in(0,1]\):

\[ Q_{n+1}=Q_n+\alpha\big[R_n-Q_n\big] \]

Unrolling this shows it is an exponential recency-weighted average — recent rewards get geometrically more weight than old ones:

\[ Q_{n+1}=(1-\alpha)^n Q_1+\sum_{i=1}^{n}\alpha(1-\alpha)^{\,n-i}R_i \]
Step sizeWeightsUse when
\(1/n\) (sample average)all rewards weighted equally; estimate convergesStationary — true values \(q_*(a)\) don’t change
constant \(\alpha\)recent rewards weighted more; never stops adaptingNon-stationary — \(q_*(a)\) drifts over time
⚙️
Significance of α, and what α = 1 does (Dec 2025 Q3b)
Larger \(\alpha\) ⇒ faster adaptation but noisier estimates. At \(\alpha=1\): \(Q_{n+1}=R_n\) — the estimate forgets everything and equals the last reward only. Great if the world changes every step; terrible if rewards are noisy, because one bad sample wipes out all history.

5. ε-greedy & the exploration/exploitation trade-off

Exploitation = pick the arm that currently looks best (greedy). Exploration = try others to improve your estimates. You cannot do both on one pull — that is the trade-off. ε-greedy resolves it with a coin flip:

Initialise Q(a)=0, N(a)=0 for all a
loop for each step:
    with probability 1-ε:  A ← argmax_a Q(a)      # exploit (greedy)
    with probability ε:    A ← a random action     # explore
    R ← reward for A
    N(A) ← N(A)+1
    Q(A) ← Q(A) + (1/N(A)) · [R − Q(A)]            # or + α[...] if non-stationary
💡
How this balances the trade-off
A fraction \(1-\varepsilon\) of pulls bank reward from the best-known arm (exploit); the remaining \(\varepsilon\) keep sampling every arm, so no arm is starved and a truly-better arm is eventually discovered. Tune \(\varepsilon\): higher = more exploration (find the best arm faster but waste more pulls); lower = more exploitation. Efficiency note: ε-greedy is simple and effective but wastes exploration uniformly — it explores even arms already known to be bad.
🎯
When pure greedy beats ε-greedy (Jul 2024-reg Q2b)
If the problem is deterministic and stationary (each arm returns a fixed reward with no noise), one pull reveals an arm’s true value, so exploration past the first round is pure waste — greedy is better. Greedy also wins when there is essentially no time left to benefit from new information (the last few pulls).

6. Two smarter exploration tricks

Optimistic Initial Values

Initialise \(Q_1(a)\) to a value higher than any plausible reward (e.g. \(Q_1=5\) when rewards are 0–3). Every arm then “disappoints” when first tried, so the agent is driven to try them all early — exploration for free, with no \(\varepsilon\).

When it worksWhen it doesn’t
Optimistic Initial ValuesStationary problems; drives early exploration without εNon-stationary problems (the optimism is a one-time effect; can’t handle change later)
UCBStationary bandits; directs exploration to genuinely uncertain armsLarge/non-stationary or full-MDP settings; costly with huge action sets

Upper Confidence Bound (UCB)

\[ A_t=\arg\max_a\Big[\,Q_t(a)+c\sqrt{\tfrac{\ln t}{N_t(a)}}\,\Big] \]

Pick the arm with the highest optimistic estimate. The bonus \(c\sqrt{\ln t/N_t(a)}\) is large for arms tried rarely (small \(N_t(a)\)) — it measures uncertainty.

💡
Significance of the confidence level c (Dec 2025 Q3c)
\(c\) sets how much you trust uncertainty over current value. Larger \(c\) ⇒ wider confidence intervals ⇒ more exploration of rarely-tried arms (e.g. give the seldom-used “Counseling call” more chances before dismissing it). Smaller \(c\) ⇒ behave more greedily on current estimates. Unlike ε-greedy’s blind randomness, UCB explores the arms it is most unsure about.

7. “Which suggestions were exploratory?” — the method

Appears on Dec 2025 Q3d, Jul 2023 Q2, Jul 2024-makeup Q3. Procedure:

  1. Before each step, list the current \(Q\)-values and find the greedy action (the arg-max).
  2. If the chosen action is not a current max ⇒ it could only have come from the ε-branch ⇒ definitely exploratory.
  3. If the chosen action is the (unique) greedy one ⇒ it was likely exploitation, but a random pick could also have landed there ⇒ possibly exploratory.
  4. If all \(Q\)-values are tied (e.g. all equal at the start) ⇒ any action is “greedy-consistent” ⇒ also only possibly exploratory.

8. Extending to many users → contextual bandits

(Jul 2023 Q2e.) When \(n\) individuals have different tastes, either keep a separate bandit per user, or use a contextual bandit where the user’s identity/features form the context and \(Q\) becomes \(Q(\text{context},a)\). Optimistic initial values help the cold-start; share learning across similar users to converge faster.
