7.5 marksQuestion 1 · Health-monitoring wristband

A wristband chooses among Mode A (low power), B (medium), C (high) each hour based on activity (resting / moderate / high).

(a) Why RL not supervised? Immediate reward vs long-term value. [1.5]

▶Show solution

Why RL. There is no labelled “correct mode” for each activity — feedback arrives only as a reward after a mode is chosen, the reward depends on the user’s (changing) activity, and good behaviour must be discovered by trying modes and observing pay-off. Supervised learning would need the correct mode supplied for every situation in advance, which we never have. So this is evaluative, trial-and-error learning ⇒ RL.

One-line distinction Immediate reward = the benefit of the mode chosen this hour; long-term value = the expected total discounted benefit from this state onward if we keep acting well.

(b) Classify each case: MAB or finite MDP? [2]

▶Show solution

Case 1 (activity independent of the mode chosen; today’s mode doesn’t affect future rewards): Multi-Armed Bandit. Each hour is an isolated choice — the action changes only the immediate reward, not the future situation, so there is effectively one state.

Case 2 (sampling Mode C too long raises battery temperature, lowering future rewards for power-hungry modes): Finite MDP. The added dependency means an action now changes future reward dynamics / state (battery temperature becomes part of the state). That coupling between present action and future return is exactly what distinguishes an MDP from a bandit.

Yes, the dependency changes the class Case 1 = bandit; introducing state-dependent future consequences (Case 2) promotes it to a finite MDP.

(c) Synchronous value iteration, γ = 0.1, one iteration, V₀ = 0. [4]

The MDP was given as a photo; here it is redrawn (black = Mode A, red = Mode B; each edge shows P, r).

FromMode A → (P, r)Mode B → (P, r)
RestingResting (1.0, 1)Resting (0.6, 1) · Moderate (0.4, 2)
ModerateModerate (0.6, 1) · Resting (0.4, 2)Moderate (0.4, 1) · High (0.6, 2)
HighHigh (0.5, 1) · Moderate (0.5, 1)High (1.0, 1)
▶Show solution

Update rule (synchronous, all \(V_0=0\)): \(V_1(s)=\max_a \sum_{s'}P(s'\mid s,a)\,[\,r+\gamma V_0(s')\,]\). Since \(V_0=0\) everywhere, the \(\gamma V_0\) terms vanish and each \(Q\) is just the expected immediate reward.

  1. Resting. A: \(1.0(1)=1.0\).   B: \(0.6(1)+0.4(2)=0.6+0.8=1.4\).  ⇒ \(V(\text{Rest})=\max(1.0,1.4)=\mathbf{1.4}\) (Mode B).
  2. Moderate. A: \(0.6(1)+0.4(2)=1.4\).   B: \(0.4(1)+0.6(2)=0.4+1.2=1.6\).  ⇒ \(V(\text{Mod})=\max(1.4,1.6)=\mathbf{1.6}\) (Mode B).
  3. High. A: \(0.5(1)+0.5(1)=1.0\).   B: \(1.0(1)=1.0\).  ⇒ \(V(\text{High})=\max(1.0,1.0)=\mathbf{1.0}\) (tie A/B).
Answer (resting, moderate, high) \(V=(\,\mathbf{1.4},\ \mathbf{1.6},\ \mathbf{1.0}\,)\). Greedy modes after iteration 1: B, B, A/B (tie).
🧠
Because \(\gamma=0.1\) is tiny and \(V_0=0\), discounting plays no role in the very first sweep — it only matters from iteration 2 onward, where successor values feed back at 10% weight.
7.5 marksQuestion 2 · ICU ventilator MDP

Adjust ventilator pressure to keep SpO₂ safe. States ≈ {lowO₂, OptimalO₂, highO₂}; actions {increase, decrease, maintain}. Increasing/decreasing leaves health unchanged w.p. 0.4; maintain always keeps the condition. Stabilises once SpO₂ is Optimal for four consecutive readings.

(a) Formulate the MDP + dynamics diagram. [2.5]

▶Show solution
  • States \(\mathcal S\): {lowO₂, OptimalO₂, highO₂} (plus an absorbing Stabilised terminal reached after 4 consecutive Optimal readings — you may track a “consecutive-optimal” count in the state).
  • Actions \(\mathcal A\): {increase pressure, decrease pressure, maintain}.
  • Transitions \(p\): increase/decrease → condition unchanged w.p. 0.4; the remaining 0.6 splits equally between the other outcomes (equal-likelihood assumption). Maintain → stays in the same state w.p. 1.
  • Rewards \(R\): positive for moving toward / staying in OptimalO₂; negative for over-ventilation (→ highO₂) and for lowO₂.
  • Discount \(\gamma\in(0,1)\); terminal = Stabilised.

(b) Reward Design A (+10 low→Optimal) vs B (−70 Optimal→high). [2]

▶Show solution

(i) Behaviour each encourages. Design A rewards reaching Optimal from a bad state but is silent about staying there, so the agent may oscillate; Design B actively punishes over-ventilation, discouraging the dangerous Optimal→high transition.

(ii) When each is preferable. Use A when the priority is recovering hypoxic (lowO₂) patients quickly; prefer B when over-ventilation is the main safety risk — the large −70 makes the agent risk-averse around the upper threshold. Preferred: Design B for patient safety (a real system would combine both: +10 to reach Optimal and −70 to deter over-ventilation).

(c) Episodic or continuing? [1.5]

▶Show solution
Episodic There is a terminal condition — four consecutive Optimal readings ⇒ the patient stabilises, ending the episode. Each patient stay is a finite episode, so the task is episodic (not endless).

(d) Impact of γ = 0.01. [1.5]

▶Show solution
Destabilising / myopic γ ≈ 0.01 makes the agent almost purely short-sighted (\(\gamma^2{=}10^{-4}\)): it chases the next-step reward and essentially ignores future consequences like sustained stabilisation, so it may take locally-rewarding but globally-unsafe actions — undermining stable long-horizon SpO₂ control.

(b) Significance of α; what if α = 1? [1.5]

▶Show solution
≤30 words α weights recent observations: larger α adapts faster to changing patient response but is noisier. At α = 1, \(Q\leftarrow r\) — e.g. Q(S) would jump straight to 8, forgetting the earlier 7 entirely; the estimate tracks only the last reward.

(c) Significance of the confidence level in UCB. [1]

▶Show solution
≤30 words The confidence term \(c\sqrt{\ln t/N(a)}\) boosts rarely-tried interventions: a higher c makes UCB explore uncertain arms (e.g. give Counseling more chances) before committing, balancing exploration against current Q-estimates.

7.5 marksQuestion 4 · Model-free control + chatbot MC

(a) Why is estimating only \(v_\pi(s)\) insufficient for control? [2]

▶Show solution

To pick the best action from state values you must evaluate \(\arg\max_a \sum_{s',r}p(s',r\mid s,a)[r+\gamma v_\pi(s')]\), which requires the model \(p(s',r\mid s,a)\). In a model-free MDP you don’t have \(p\), so even a perfect \(v_\pi\) cannot tell you which action leads to those good successor states. You therefore estimate action values \(q_\pi(s,a)\), because \(\arg\max_a q_\pi(s,a)\) selects actions without any model.

(b) Why can first-visit MC under a deterministic policy fail? Fix? [2]

▶Show solution

A deterministic policy always takes the same action in a state, so the other actions are never visited and their \(q\)-values are never updated from their initial guesses. The agent can never discover that an unexplored action is better, so it can lock onto a sub-optimal policy. Fix: ensure continual exploration — use an ε-soft policy (every action has probability ≥ ε/|𝒜|) or exploring starts.

Convention note
Here each visited pair’s estimate is set to its first observed return (textbook “first return = first average”). If your instructor instead treats the given Q as a prior sample, average it with the return — e.g. Q(s₁,a₂)=(0.5+2.2)/2=1.35. Either way the greedy action and policy change are identical (s₁: a₃→a₂), so state your assumption and the conclusion holds. ε itself isn’t given; any ε∈(0,1) works — ε=0.3 just gives clean 0.8 / 0.1 / 0.1.
