1. The Monte Carlo idea

If value is “expected return”, just play out whole episodes and average the returns. No model, no bootstrapping — only real, sampled experience.

\[ v_\pi(s)\approx \text{average of returns } G_t \text{ observed after visits to } s \]
💡
MC vs. TD vs. DP in one line
DP: bootstrap + model (no sampling). TD: bootstrap + sample (update every step). MC: sample + no bootstrap — wait until the episode ends, then use the actual return. MC therefore needs episodic tasks.

2. First-visit vs. every-visit

First-visit MCEvery-visit MC
Average the return following only the first time \(s\) is visited in each episode.Average the return following every visit to \(s\), even repeats within an episode.
Returns are independent ⇒ unbiased; classic convergence.Returns within an episode are correlated; also converges (biased for finite data but consistent).
⚡
Which converges faster to \(v_*\)? (Jul 2023 Q4a)
Every-visit uses more data points per episode, so with few episodes it often produces estimates sooner; first-visit gives cleaner, unbiased estimates. Both converge to the true value as episodes → ∞. In exam answers, note the data-efficiency vs. bias trade-off.

4. Why estimating \(v_\pi\) alone is insufficient for control

🚫
The model-free control problem (Dec 2025 Q4a)
To act greedily from \(v_\pi\) you’d need \(\arg\max_a\sum_{s',r}p(s',r\mid s,a)[r+\gamma v_\pi(s')]\) — which requires the model \(p\). In a model-free setting you don’t have \(p\), so a perfect \(v_\pi\) still doesn’t tell you which action is best. The fix: estimate action values \(q_\pi(s,a)\) directly — then \(\arg\max_a q_\pi(s,a)\) needs no model at all.

5. Keeping exploration alive

MC learns about actions only by trying them. A deterministic greedy policy may never try some \((s,a)\) pairs, so their values are never learned and the policy can get stuck. Two fixes (Jul 2023 Q4c):

Fix 1

Exploring Starts

Begin episodes from a random \((s,a)\) pair so every action in every state has a chance to be the starting move. Guarantees coverage; impractical if you can’t choose the start.

Fix 2

ε-soft policies

Use a policy that gives every action probability \(\ge \varepsilon/|\mathcal A|>0\) (e.g. ε-greedy). No special starts needed — exploration is baked into the behaviour.

⚠️
Why first-visit MC under a deterministic policy can fail (Dec 2025 Q4b)
A deterministic policy always picks one action per state, so the alternative actions are never visited — their \(q\)-values stay at their initial estimates and can never be shown to be better. The policy can lock onto a sub-optimal action forever. Fix: make the policy ε-soft (or use exploring starts) so every action keeps getting sampled.

6. On-policy first-visit MC control for ε-soft policies

Initialise Q(s,a)=0, Returns(s,a)=[] , π = arbitrary ε-soft
for each episode:
    generate S0,A0,R1,...,ST  following π
    G ← 0
    for t = T-1 down to 0:
        G ← γG + R_{t+1}
        if (S_t,A_t) is a FIRST visit:
            append G to Returns(S_t,A_t)
            Q(S_t,A_t) ← average(Returns(S_t,A_t))
            A* ← argmax_a Q(S_t,a)              # greedy action
            for all a:  π(a|S_t) ← { 1-ε+ε/|A|  if a=A*
                                     ε/|A|        otherwise }
💡
ε-soft ⊃ ε-greedy
An ε-soft policy is any policy with \(\pi(a\mid s)\ge \varepsilon/|\mathcal A|\) for all \(a\). ε-greedy is the specific ε-soft policy that puts all remaining mass on the greedy action: greedy gets \(1-\varepsilon+\varepsilon/|\mathcal A|\), the others \(\varepsilon/|\mathcal A|\) each. So every ε-greedy policy is ε-soft, but not vice-versa.

8. Off-policy learning & importance sampling

Learn about a target policy \(\pi\) while behaving with a different behaviour policy \(b\) (which explores). Reweight returns by the importance-sampling ratio:

\[ \rho_{t:T-1}=\prod_{k=t}^{T-1}\frac{\pi(A_k\mid S_k)}{b(A_k\mid S_k)} \]
🔗
Ratio vs. model dynamics (Jul 2023 Q4b)
Notice the environment’s transition probabilities \(p(s'\mid s,a)\) appear in both the numerator and denominator of the trajectory probability and cancel. So \(\rho\) depends only on the two policies, never on the model — which is exactly why off-policy MC is model-free.
📏
Coverage requirement (Jul 2023 Q4d)
Yes — coverage is mandatory. The behaviour policy \(b\) must give non-zero probability to every action the target \(\pi\) might take: if \(\pi(a\mid s)>0\) then \(b(a\mid s)>0\). Otherwise \(\rho\) divides by zero — you’d be trying to learn about actions you never sample. (\(b\) may, however, take actions \(\pi\) never would.)

9. Check yourself

Q1. MC methods require…
a known model of the environment
complete episodes (terminating tasks)
bootstrapping from current estimates
a continuous action space
MC averages full returns, so an episode must end before its return is known. It is model-free and does not bootstrap.
Q2. In model-free control we estimate \(q_\pi(s,a)\) rather than \(v_\pi(s)\) because…
choosing the best action from \(v\) would need the model
\(v\) cannot be estimated by MC
\(q\) needs no episodes
\(v\) ignores rewards
\(\arg\max_a q(s,a)\) is model-free; doing the same from \(v\) requires \(p(s',r|s,a)\), which we don’t have.
Q3. An ε-greedy policy with 4 actions assigns the greedy action probability…
\(1-\varepsilon+\varepsilon/4\)
\(1-\varepsilon\)
\(\varepsilon/4\)
\(1\)
Greedy gets \(1-\varepsilon\) plus its share \(\varepsilon/|\mathcal A|\) of the random branch; each non-greedy action gets \(\varepsilon/4\).
